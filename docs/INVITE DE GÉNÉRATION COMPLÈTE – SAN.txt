INVITE DE GÉNÉRATION COMPLÈTE – SANKOFA

Tu es un architecte logiciel et développeur full-stack expert. Génère l’intégralité du code source de l’application **SANKOFA**, une marketplace de vente et location d’œuvres d’art africaines, selon les spécifications ci-dessous.

## Stack technique obligatoire
- Frontend : Next.js 14 (App Router) + TypeScript + Tailwind CSS (PWA)
- Backend : API routes Next.js + Supabase Edge Functions
- Base de données : Supabase (PostgreSQL) avec RLS et schéma fourni
- Authentification : Supabase Auth (magic link / OTP téléphone)
- Paiement : Paynote (Mobile Money) + Stripe (cartes) – mode sandbox
- Blockchain : Polygon (testnet Mumbai) via ethers.js (certificats d’authenticité)
- Stockage : Supabase Storage (photos, PDF)
- Notifications : Email (Resend) + WhatsApp (Twilio) + PWA push
- Logging : Sentry + Logtail (fichiers de config)
- Hébergement : Vercel (avec configuration adaptée)

## Structure du projet à générer
- Dossier `src/` avec :
  - `app/` (pages : accueil, catalogue, fiche œuvre, panier, paiement, dashboard artiste, espace acheteur, admin, location)
  - `components/ui/` (composants réutilisables : Button, Card, Modal, FormField, Stepper…)
  - `lib/` (supabase client, paynote client, blockchain, utils)
  - `hooks/` (useAuth, useCart, useRental, etc.)
  - `types/` (interfaces TypeScript)
- Fichiers de configuration : `tailwind.config.js` (avec les couleurs et tokens fournis), `next.config.js` (PWA), `vercel.json`, `supabase/migrations/` (schéma SQL)
- Tests : `__tests__/` (Jest, Supertest, Cypress)
- Documentation : `README.md` (installation, variables d’environnement, déploiement)

## Schéma de base de données (PostgreSQL – extrait)

## Fonctionnalités à implémenter (MVP + location V2)

### Côté public (sans login)
- Page d’accueil avec grille d’œuvres (filtres: style, pays, prix, disponibilité location/achat)
- Recherche plein texte
- Fiche œuvre détaillée (carrousel photos, vidéo, description, artiste, prix, option location)
- Inscription / connexion (email/téléphone + OTP)

### Côté acheteur (connecté)
- Panier (ajout/suppression, calcul frais livraison)
- Paiement par Mobile Money (Orange Money, MTN MoMo) via Paynote – avec séquestre (escrow)
- Paiement par carte (Stripe)
- Suivi de commande (timeline : payé → expédié → livré)
- Délai de rétractation 48h (bouton « Signaler un problème »)
- Avis et notation des artistes
- Espace « Mes locations » (V2) : choix durée (1/3/6 mois), option d’achat avec déduction des loyers

### Côté artiste (connecté)
- Inscription avec vérification (upload CNI + selfie) – validation admin
- Académie SANKOFA (3 modules obligatoires : photographie, emballage, description) avec quiz
- Dépôt d’œuvre (formulaire : titre, description, dimensions, matériaux, prix, photos, option location)
- Tableau de bord : ventes, vues, gains, liste des œuvres, commandes à expédier
- Génération d’étiquette d’expédition (PDF) et confirmation d’envoi
- Réception des paiements sur Mobile Money après validation acheteur

### Côté administrateur
- Validation des inscriptions artistes (documents)
- Validation des œuvres (refus possible avec motif)
- Gestion des litiges (médiation, remboursement partiel/total)
- Tableau de bord KPIs (GMV, nombre de commandes, taux de litiges, etc.)

### Fonctionnalités transverses
- Certificat d’authenticité : généré automatiquement après validation acheteur (PDF avec QR code), hash stocké sur Polygon (testnet)
- Notifications : email + WhatsApp pour événements (vente, expédition, litige)
- PWA : manifest, service worker, notifications push

- accent: '#D9A13B' (or ocre)

Typographie : titres en Playfair Display, corps en Inter.
Composants : Button (primary, secondary, danger), Card, Modal, FormField, Stepper, ArtworkCard, RentalOptionCard.
Accessibilité : WCAG AA (contraste, focus visible, cibles ≥44px).

- Aucune clé API en clair dans le code (utilisation des variables d’environnement).

- Validation des entrées utilisateur côté serveur.
- Rate limiting sur les routes sensibles (paiement, inscription).

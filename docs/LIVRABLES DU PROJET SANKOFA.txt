## 1. PRD.md – Product Requirements Document

```markdown
# PRD – SANKOFA

## 1. Vision
Créer la première marketplace panafricaine dédiée aux artistes émergents, permettant d’acheter et de vendre des œuvres d’art originales en ligne avec séquestre des fonds, livraison sécurisée, et certificat d’authenticité blockchain.

## 2. Objectifs (SMART)
- **Artistes** : 100 artistes actifs au Cameroun dans les 6 mois post‑MVP.
- **Ventes** : 100 transactions par mois à 6 mois.
- **Délai de livraison** : < 5 jours ouvrés pour les livraisons nationales.
- **Litiges** : Taux < 5% des transactions.

## 3. Périmètre fonctionnel (MVP)

### 3.1. Artistes
- Inscription avec vérification d’identité (CNI + selfie avec œuvre)
- Dépôt d’œuvre (titre, description, dimensions, matériaux, prix, 3‑5 photos HD)
- Tableau de bord (statistiques de vues, commandes, gains)
- Préparation d’expédition (génération d’étiquette, confirmation d’envoi)
- Réception des fonds sur Mobile Money (Orange Money, MTN MoMo)

### 3.2. Acheteurs
- Inscription par email/téléphone
- Navigation, filtres (prix, pays, style, matériaux)
- Paiement sécurisé par Mobile Money ou carte bancaire (Stripe)
- Suivi de livraison en temps réel
- Délai de rétractation / inspection : 48h après réception
- Avis et notation des artistes

### 3.3. Modérateurs / Admins
- Validation des artistes (documents d’identité)
- Validation des œuvres (qualité photos, conformité éditoriale)
- Gestion des litiges (médiation, remboursement partiel/total)
- Tableau de bord KPIs (ventes, nombre d’utilisateurs, litiges)

### 3.4. Certificat d’authenticité
- Généré automatiquement après validation acheteur (PDF + QR code)
- Enregistrement sur Polygon (hash de l’œuvre + métadonnées)
- Page de vérification publique accessible par QR code

### 3.5. Logistique
- Hub central à Douala (optionnel en phase 1 : expédition directe artiste → acheteur)
- Tracking API via partenaire transporteur (ex: Creseada, DHL)
- Assurance transport incluse dans les frais de livraison

## 4. Contraintes critiques
- **Paiement séquestre** : les fonds sont bloqués jusqu’à validation de l’acheteur ou expiration du délai de 48h.
- **Conformité douanière** : pour les expéditions internationales, intégration des procédures NCMM (Nigeria) et autorisations ivoiriennes.
- **Protection des données** : conformité RGPD et loi camerounaise n°2010/012.
- **Droit d’auteur** : l’artiste conserve tous ses droits. Plagiat interdit.

## 5. Métriques de succès (KPIs)
- Taux de conversion (visite → achat) : > 3%
- Panier moyen : > 100 000 FCFA
- Délai médian de livraison : < 7 jours
- NPS (satisfaction acheteur) : > 40
- Taux de litiges : < 5%
```

# Architecture technique – SANKOFA

## 1. Vue d’ensemble

```
[Client Web (Next.js)] 
      │
      ▼
[Vercel (frontend + API routes)]
      │
      ├──► [Supabase (auth, DB, storage, realtime)]
      ├──► [Paynote API (Mobile Money)]
      ├──► [Stripe API (cards)]
      ├──► [Transporteur API (tracking)]
      ├──► [Deepseek API (assistant IA)]
      └──► [Polygon blockchain (certificats)]

## 2. Stack détaillée

| Couche | Technologie | Rôle |
|--------|-------------|------|
| Frontend | Next.js 14 (App Router) + TypeScript + Tailwind CSS | UI responsive, PWA |
| Prototypage UI | Google Stitch (MCP) | Génération rapide de composants |
| Backend / API | Next.js API routes (serverless) | Logique métier, webhooks |
| Base de données | Supabase (PostgreSQL) | Utilisateurs, œuvres, commandes, escrow |
| Authentification | Supabase Auth (magic link / OTP) | Connexion par email/téléphone |
| Stockage fichiers | Supabase Storage | Photos des œuvres, PDF certificats |
| Realtime | Supabase Realtime | Notifications (vente, livraison) |
| Paiement Mobile Money | Paynote (agrégateur) | Intégration Orange Money, MTN MoMo |
| Paiement carte | Stripe | Acheteurs internationaux |
| Blockchain | Polygon (via Thirdweb) | Enregistrement hash certificats |
| Assistant IA | Deepseek API | Chatbot, génération descriptions, modération |
| Logistique | API partenaire (Creseada, etc.) | Suivi colis, étiquettes |
| Hébergement | Vercel (Pro plan) | Frontend + serverless functions |
| CI/CD | GitHub Actions | Tests, déploiement automatique |

## 3. Modèle de données clés (PostgreSQL)

```sql
-- Extraits
users (id, email, phone, role, verified_at, kyc_docs)
artworks (id, artist_id, title, description, price, dimensions, materials, status, photos[])
orders (id, buyer_id, artwork_id, amount, escrow_status, delivery_status, created_at)
escrow_transactions (id, order_id, amount, status, released_at, dispute_reason)
certificates (id, artwork_id, blockchain_tx_hash, qr_code_url)
litigations (id, order_id, requester, reason, resolution, resolved_at)
```

## 4. Flux de séquestre (escrow)

1. Acheteur initie paiement → Paynote capture les fonds vers un compte séquestre technique.
2. Webhook Paynote met à jour `orders.escrow_status = 'pending'`.
3. Artiste confirme expédition → status = 'shipped'.
4. Acheteur valide réception (ou silence 48h) → status = 'released'.
5. Déclenchement virement vers Mobile Money de l’artiste via Paynote Payout.

## 5. Sécurité et conformité
- Chiffrement TLS 1.3.
- Variables d’environnement sur Vercel (clés API, secrets).
- Row Level Security (RLS) sur Supabase.
- Logs d’audit pour actions admin (modifications, remboursements).
- Respect des lois douanières via module séparé (fiche pays).

## 6. Déploiement (GitHub Actions + Vercel)
- Branch `main` → déploiement automatique en production.
- Pull requests → preview Vercel + tests automatisés.
- Migrations Supabase exécutées manuellement (ou via script CI).
```

### User journey – Acheteur

1. Découvre l’œuvre sur Instagram / recherche Google → clique sur lien SANKOFA.
2. Navigue, filtre par style « abstrait » et prix < 200 000 FCFA.
3. Consulte la fiche, zoom sur les photos, lit la description générée par Deepseek.
4. Clique « Acheter » → panier → choisit Mobile Money (Orange Money).
5. Saisit son numéro → autorise le paiement sur son téléphone.
6. Reçoit un SMS : « Paiement accepté, fonds séquestrés. L’artiste va expédier. »
7. Suit la livraison via le lien de tracking.
8. Reçoit l’œuvre, l’inspecte, valide dans son espace client (ou ne fait rien, déblocage auto après 48h).
9. Reçoit par email le certificat d’authenticité PDF avec QR code.
10. Laisse un avis étoilé sur l’artiste.

## Préambule

La Plateforme exploite une marketplace en ligne destinée à la vente d’œuvres d’art originales par des artistes africains, avec séquestre des fonds, logistique et certification blockchain. L’Artiste souhaite proposer ses œuvres à la vente via la Plateforme.

Les parties conviennent de ce qui suit :

---

## Article 1 – Objet

Le présent contrat a pour objet de définir les conditions d’inscription, de mise en vente, de vente, de livraison et de rémunération des œuvres de l’Artiste sur la Plateforme SANKOFA.

---

## Article 2 – Adhésion et vérification d’identité

2.1. L’Artiste déclare être majeur (18 ans révolus) ou représenté par un parent tuteur.  
2.2. L’Artiste s’engage à fournir une pièce d’identité valide (CNI, passeport) et un selfie tenant une de ses œuvres. Ces documents sont conservés conformément à la politique de confidentialité.  
2.3. La Plateforme se réserve le droit de refuser une adhésion sans motif, notamment en cas de suspicion de fraude ou d’antécédent de contrefaçon.

---

## Article 3 – Dépôt et validation des œuvres

3.1. L’Artiste dépose chaque œuvre via son espace personnel en fournissant :  
- Titre, dimensions, matériaux, année de création, description (histoire, technique, message), prix en FCFA,  
- 3 à 5 photos haute résolution (≥ 800×600 px), fond neutre,  
- (optionnel) une courte vidéo de présentation.

3.2. La Plateforme peut refuser la publication d’une œuvre si :
- les photos sont de qualité insuffisante,
- l’œuvre est une copie ou contrevient au droit d’auteur,
- l’œuvre a été générée entièrement par une IA (sauf mention expresse),
- elle contient des propos ou symboles illégaux ou contraires aux bonnes mœurs.

3.3. L’Artiste certifie être l’auteur original de l’œuvre et en détenir tous les droits patrimoniaux.

---

## Article 4 – Prix, commission et paiement

4.1. **Prix de vente** : fixé librement par l’Artiste en FCFA, hors frais de livraison et hors commission.

4.2. **Commission de la Plateforme** :
- 25 % du prix de vente pour la **première vente** de l’Artiste,
- 15 % pour les **ventes suivantes**.
La commission est prélevée automatiquement avant le reversement à l’Artiste.

4.3. **Séquestre (escrow)** :
- À l’achat, les fonds sont bloqués sur un compte séquestre technique.
- Ils sont débloqués automatiquement 48 heures après la réception validée par l’acheteur (ou au terme des 48h sans réclamation).
- Le délai de rétractation / inspection est de 48h à compter de la livraison.

4.4. **Paiement à l’Artiste** :
- Le montant net (prix – commission) est viré sur le compte Mobile Money (Orange Money, MTN MoMo) déclaré par l’Artiste, dans les 24h suivant le déblocage.
- Seuil minimum de virement : 10 000 FCFA.

---

## Article 5 – Obligations de l’Artiste

5.1. **Emballage et expédition** :  
- L’Artiste s’engage à expédier l’œuvre dans les 3 jours ouvrés suivant la notification de vente.  
- L’emballage doit être professionnel (caisse rigide, matériaux de protection, pas de ruban adhésif sur l’œuvre).  
- Un guide d’emballage est fourni par la Plateforme.

5.2. **Assurance** :  
- La Plateforme souscrit une assurance transport pour chaque envoi. L’Artiste doit déclarer immédiatement tout dommage constaté lors de l’emballage.

5.3. **Droit d’auteur** :  
- L’Artiste reste titulaire de ses droits. Il autorise la Plateforme à reproduire les images des œuvres pour la promotion sur le site et les réseaux sociaux, sans cession exclusive.

5.4. **Interdictions** :  
- Il est interdit de vendre des œuvres contrefaisantes, volées, ou issues d’espèces protégées (CITES) sans permis.  
- L’usage d’IA générative doit être mentionné dans la description.

---

## Article 6 – Certificat d’authenticité

6.1. Pour chaque vente finalisée (après validation acheteur), la Plateforme génère un certificat d’authenticité numérique (PDF) avec un QR code et enregistre une empreinte (hash) sur la blockchain Polygon.

6.2. L’Artiste reconnaît que ce certificat est émis par la Plateforme sur la base des informations qu’il a fournies.

---

## Article 7 – Litiges

7.1. En cas de non-conformité ou de dommage, l’acheteur dispose d’un bouton « Signaler un problème » dans son espace.

7.2. La Plateforme agit en médiateur. Elle peut :
- demander le retour de l’œuvre aux frais de la Plateforme,
- proposer un remboursement partiel,
- débloquer les fonds à l’Artiste si le litige est infondé.

7.3. L’Artiste s’engage à collaborer de bonne foi (fournir des photos, accepter un retour si nécessaire).

---

## Article 8 – Résiliation

8.1. L’Artiste peut résilier son compte à tout moment, sous réserve d’avoir honoré les commandes en cours.

8.2. La Plateforme peut résilier le contrat sans préavis en cas de :
- fraude, contrefaçon avérée,
- non-respect des délais d’expédition répété,
- diffusion de fausses informations.

---

## Article 9 – Responsabilité

9.1. La Plateforme n’est pas responsable des retards de livraison imputables au transporteur, sauf si elle a choisi ce transporteur.

9.2. En cas de perte ou de casse pendant le transport, l’assurance souscrite par la Plateforme indemnise l’Artiste sur la base du prix de vente net (après commission). L’Artiste ne peut réclamer davantage.

9.3. La Plateforme ne garantit pas que chaque œuvre trouvera un acheteur.

---

## Article 10 – Confidentialité et données personnelles

Les données de l’Artiste sont traitées conformément à la politique de confidentialité de la Plateforme. L’Artiste consent à ce que ses informations (nom, ville, biographie) soient visibles publiquement sur son profil.

---

## Article 11 – Droit applicable et juridiction compétente

Le présent contrat est régi par la loi camerounaise. Tout litige relatif à son interprétation ou à son exécution sera soumis au tribunal de première instance de Douala, sauf disposition impérative contraire.

# Guide d’utilisation – SANKOFA

## Pour les artistes

### 1. Créer son compte
- Allez sur `SANKOFA.com/inscription`
- Choisissez « Je suis artiste »
- Saisissez votre email et numéro de téléphone
- Validez le code reçu par SMS
- Téléchargez une photo de votre pièce d’identité + un selfie tenant une de vos œuvres
- Attendez la validation (max 48h)

### 2. Déposer une œuvre
- Connectez-vous → tableau de bord → « Déposer une œuvre »
- Remplissez : titre, description (histoire, technique), dimensions, matériaux, prix
- Téléchargez 3 à 5 photos (fond neutre, bonne lumière)
- Soumettez à validation

### 3. Quand une œuvre est vendue
- Vous recevez un email et un WhatsApp : « Félicitations, votre œuvre [titre] a été vendue »
- Préparez l’œuvre : emballage rigide (voir guide emballage)
- Générez l’étiquette depuis votre tableau de bord
- Déposez le colis chez le transporteur ou en point relais
- Saisissez le numéro de suivi

### 4. Recevoir le paiement
- L’acheteur a 48h après réception pour valider
- Une fois validé (ou délai passé), les fonds sont virés sur votre Mobile Money sous 24h (commission déduite)

### 5. Certificat d’authenticité
- Après chaque vente, un certificat PDF est généré et un hash est stocké sur blockchain
- Vous pouvez le télécharger depuis votre historique

## Pour les acheteurs

### 1. Acheter une œuvre
- Parcourez le catalogue, utilisez les filtres (prix, pays, style)
- Cliquez sur une œuvre → « Ajouter au panier »
- Renseignez votre adresse de livraison
- Choisissez Mobile Money (Orange ou MTN) ou carte bancaire
- Validez le paiement : les fonds sont bloqués (séquestre)

### 2. Suivre la livraison
- Recevez un email avec le lien de suivi
- Vous pouvez consulter le statut dans votre espace « Mes commandes »

### 3. À la réception
- Inspectez l’œuvre dans les 48h
- Si conforme : cliquez « Réception conforme » → l’artiste est payé, vous recevez le certificat
- Si problème (casse, non-conformité) : cliquez « Signaler un problème » et joignez des photos

### 4. Certificat d’authenticité
- Téléchargez le PDF depuis l’historique de commande
- Scannez le QR code pour vérifier l’authenticité en ligne

## En cas de litige
- Contactez le support via WhatsApp (`+237 XXXX`) ou email `support@SANKOFA.com`
- Un médiateur examinera votre dossier dans les 72h
```

# PILIER 1 – STACK TECHNIQUE UNIFIÉE

## 1.1. Frontend

| Élément | Choix | Détails |
|---------|-------|---------|
| Framework | Next.js 14 (App Router) | Rendu côté serveur (SSR) pour le SEO des fiches œuvres, génération statique pour les pages d’accueil et catalogue. |
| Langage | TypeScript (strict mode) | Typage des props, des réponses API, réduction des bugs de runtime. |
| Styling | Tailwind CSS + composants personnalisés | Utilitaire-first, responsive, personnalisation via `tailwind.config.js` (couleurs terracotta, beige, vert forêt). |
| Gestion d’état | Context API + `useReducer` (pas de Redux) | Pas besoin de bibliothèque lourde ; état global pour le panier, les filtres, l’utilisateur. |
| PWA | Manifest + Service Worker | Installation sur mobile, notifications push, cache du catalogue en lecture seule. |
| Appels API | `@supabase/supabase-js` + fetch natif | Auth, base de données, storage ; les appels à Paynote/Stripe via API routes. |
| SEO | Métadonnées dynamiques, sitemap, robots.txt | Chaque œuvre a sa propre page avec titre, description, balises Open Graph. |

## 1.2. UI Prototyping

| Outil | Rôle | Workflow |
|-------|------|----------|
| Google Stitch (MCP) | Génération rapide de maquettes Next.js/Tailwind | Prompt en langage naturel → code React exportable. Utilisation en phase de conception avec l’agent Antigravity. |
| Figma | Refinement et design system | Import du code Stitch, création de composants réutilisables (boutons, cartes œuvre, formulaires). |

## 1.3. Backend & API

| Composant | Choix | Justification |
|-----------|-------|----------------|
| API routes | Next.js (serverless) | Hébergées sur Vercel, sans serveur dédié, scaling automatique. |
| Edge Functions | Supabase Edge Functions | Pour les webhooks (Paynote, Stripe) et les traitements sensibles (séquestre). |
| Base de données | Supabase PostgreSQL | Tables : users, artists, artworks, orders, escrow_transactions, certificates, litigations, reviews. |
| Row Level Security (RLS) | Activé sur toutes les tables | Politiques : artiste peut voir/modifier ses propres œuvres, acheteur ne voit que ses commandes, admin tout. |
| Stockage fichiers | Supabase Storage | Buckets : `artwork-photos` (images publiques), `id-documents` (privé), `certificates-pdf` (public après vente). |
| Realtime | Supabase Realtime | Notifications de changement de statut commande, chat de support. |

## 1.4. Paiement et séquestre

| Service | Rôle | Intégration |
|---------|------|--------------|
| Paynote | Agrégateur Mobile Money (Orange Money, MTN MoMo) | API REST : collecte, séquestre, payout. Webhooks pour statut de paiement. |
| Stripe Connect | Cartes bancaires (acheteurs internationaux) | Paiement en FCFA converti, séquestre géré par Stripe (marketplace standard). |
| Compte séquestre technique | Compte Paynote dédié | Les fonds sont bloqués jusqu’à validation acheteur. |

**Flux de séquestre détaillé :**
1. Acheteur initie paiement → Paynote collecte et bloque les fonds sur le compte séquestre.
2. Webhook Paynote `payment.succeeded` → mise à jour `orders.escrow_status = 'pending'`.
3. Artiste expédie → changement `orders.delivery_status = 'shipped'`.
4. Acheteur valide (ou délai 48h) → webhook déclenché.
5. Appel à l’API Paynote payout → virement net vers Mobile Money de l’artiste.
6. En cas de litige, les fonds restent bloqués jusqu’à décision admin.

## 1.5. Blockchain (certificat d’authenticité)

| Composant | Choix | Détails |
|-----------|-------|---------|
| Réseau | Polygon (Mainnet) | Frais de gaz très bas (< 0,01 $). Utilisation du testnet Mumbai pour les tests. |
| Stockage hash | Smart contract unique | Fonction `storeArtHash(bytes32 hash, string memory metadataURI)` émettant un event. |
| Metadata | Lien vers JSON hébergé sur Supabase Storage | Contient : id œuvre, titre, artiste, dimensions, date vente, URL certificat PDF. |
| Interface | `ethers.js` v6 | Connexion à un nœud Infura ou Alchemy. Clé privée du wallet de la plateforme (sécurisée dans Vercel env). |
| Gas | Payé par la plateforme | Budget mensuel 10 $ pour les premières milliers de transactions. |

## 1.6. Logistique

| Phase | Solution | Intégration |
|-------|----------|--------------|
| Phase 1 (MVP) | Expédition directe artiste → acheteur, code de suivi fourni par transporteur (DHL, Creseada). | Artiste saisit manuellement le numéro de suivi ; API DHL pour tracking optionnelle. |
| Phase 2 (hub central) | Hub à Douala : réception, inspection, réexpédition. | Logiciel interne (Next.js) pour scan de codes-barres, impression étiquette. |
| Assurance | Police cadre avec partenaire assurance (ex: AXA Cameroun). | Coût inclus dans les frais de livraison (forfait 5 000 FCFA par colis). |

## 1.7. IA / Assistant

| Usage | Modèle | Intégration |
|-------|--------|--------------|
| Chatbot support | Deepseek API (version légère) | Embedding sur la page d’aide, répond aux questions fréquentes (délais, litiges, paiement). |
| Génération description œuvre | Deepseek API | L’artiste peut cliquer « Générer une description » à partir du titre et des mots-clés ; il modifie avant soumission. |
| Modération de contenu | Deepseek API + règles locales | Détection de contenu offensant dans les descriptions et les avis. |

## 1.8. Hébergement & CI/CD

| Service | Rôle | Configuration |
|---------|------|----------------|
| Vercel (Pro) | Hébergement Next.js | Branche `main` → production, `develop` → preview. Limite de fonctions : 60s timeout, adapté à nos API routes. |
| Supabase Cloud | Base de données, storage, edge functions | Plan Pro (25 $/mois) pour RLS, backups quotidiens. |
| GitHub Actions | CI/CD | Workflows : `test.yml` (Jest, Cypress), `deploy-staging.yml` (déploiement auto sur Vercel staging). |
| Monitoring | Sentry + Logtail | Sentry capture les erreurs frontend/backend ; Logtail centralise les logs des edge functions. |

## 5.2. Fonds de garantie (pour litiges)

- **Alimentation** : 1 % de chaque vente est prélevé et placé sur un compte dédié.
- **Utilisation** : Remboursement rapide des acheteurs en cas de litige avéré, avant recours contre l’artiste.
- **Montant cible** : 2 000 000 FCFA (atteint après environ 200 ventes à 150 000 FCFA de panier moyen).

## 5.3. Assurance transport (police cadre)

- **Souscripteur** : SANKOFA pour le compte des artistes.
- **Couverture** : Vol, perte, casse totale ou partielle.
- **Indemnisation** : Valeur déclarée (prix de vente net après commission), déduction d’une franchise de 10 %.
- **Procédure** : L’acheteur signale dans les 48h, fournit photos, la plateforme déclare sinistre.

# Modèle économique détaillé – SANKOFA

## 1. Vue d’ensemble

SANKOFA est une **marketplace bilatérale** connectant artistes africains et acheteurs (particuliers, collectionneurs, entreprises). Le modèle économique repose sur des **commissions sur les ventes**, des **services optionnels** et des **partenariats**. La stratégie est d’être accessible aux artistes émergents (coût d’entrée nul ou très faible) tout en générant des revenus suffisants pour couvrir les coûts et assurer la rentabilité à moyen terme.

---

## 2. Sources de revenus

| Source | Description | Tarification | Marge brute estimée |
|--------|-------------|--------------|---------------------|
| **Commission sur vente (artistique)** | Prélèvement sur le prix de l’œuvre lors de chaque transaction. | 25 % sur la **première vente** de l’artiste, **15 %** sur les suivantes. Commission calculée hors frais de livraison. | 100 % (pas de coût direct) |
| **Frais de mise en avant** | Option pour les artistes : meilleure visibilité (page d’accueil, newsletter, badge “coup de cœur”). | Forfait mensuel : 20 000 FCFA. Engagement min. 3 mois. | 100 % |
| **Commission curateur** | Prélèvement sur les ventes générées par une exposition virtuelle (payé par la plateforme, pas par l’artiste). | 5 % du prix de vente (prélevé sur la commission plateforme). | 0 % pour la plateforme (redistribution) |
| **Frais de livraison (optional)** | Marge sur les frais de transport facturés à l’acheteur. | Frais réels + 10 % de marge. | 10 % |

| **Abonnement galerie virtuelle** | Pour les galeristes souhaitant un espace de marque dédié avec analytics avancés. | 100 000 FCFA/mois. | 100 % |
| **Données agrégées (anonymisées)** | Rapports de marché pour institutions, investisseurs, chercheurs. | Sur devis (ex: 500 000 FCFA/rapport). | 100 % |

## 4. Analyse de rentabilité (scénario à 12 mois)

### Hypothèses de volume (croissance progressive)

| Mois | Commandes/mois | Panier moyen (FCFA) | Commission moyenne* |
|------|----------------|---------------------|---------------------|
| 1-3 | 10 | 150 000 | 25 % (premières ventes) |
| 4-6 | 30 | 150 000 | 20 % (mélange 1ère et suivantes) |
| 7-9 | 80 | 150 000 | 18 % |
| 10-12 | 150 | 150 000 | 15 % (artistes établis) |

*Commission moyenne pondérée (25 % pour les nouvelles œuvres, 15 % pour les artistes ayant déjà vendu)

### Calcul des revenus annuels

| Période | Commandes | Commission moyenne | Revenu commission | Autres revenus* | Total |
|---------|-----------|-------------------|-------------------|----------------|-------|
| Mois 1-3 | 30 | 25 % | 30 × 150k × 0,25 = 1 125 000 | 100 000 | 1 225 000 |
| Mois 4-6 | 90 | 20 % | 90 × 150k × 0,20 = 2 700 000 | 300 000 | 3 000 000 |
| Mois 7-9 | 240 | 18 % | 240 × 150k × 0,18 = 6 480 000 | 600 000 | 7 080 000 |
| Mois 10-12 | 450 | 15 % | 450 × 150k × 0,15 = 10 125 000 | 900 000 | 11 025 000 |
| **Total année** | **810** | | **20 430 000** | **1 900 000** | **22 330 000** |

*Autres revenus : mises en avant, assurance premium, NFT, abonnements (estimation basse).

### Coûts annuels

| Catégorie | Montant (FCFA) |
|-----------|----------------|
| Coûts fixes (12 mois) | 1 928 000 × 12 = 23 136 000 |
| Coûts variables (810 commandes × 2 500 FCFA) | 2 025 000 |
| Coûts de développement (amortis sur 12 mois) | 12 500 000 (cf budget) |
| **Total coûts** | **37 661 000** |

**Résultat net** = 22 330 000 – 37 661 000 = **-15 331 000 FCFA** (perte la première année).

### Seuil de rentabilité

Le seuil de rentabilité est atteint lorsque le revenu mensuel dépasse les coûts fixes + variables.  
Coûts fixes mensuels ~1 930 000 FCFA.  
Marge brute moyenne par commande : prix 150 000 FCFA, commission 18 % = 27 000 FCFA, moins coûts variables 2 500 FCFA = 24 500 FCFA de contribution.

**Seuil** = 1 930 000 / 24 500 ≈ **79 commandes/mois**.

→ Avec un panier moyen de 150 000 FCFA, le seuil de rentabilité est atteint à environ **80 commandes par mois** (soit environ 12 millions de FCFA de GMV mensuel).

## 5. Stratégie de pricing et positionnement

### Pour les artistes
- **Commission dégressive** : incite l’artiste à rester sur la plateforme et à vendre régulièrement.
- **Aucun frais d’inscription** : barrière d’entrée nulle.
- **Paiement rapide** (24h après validation acheteur) : facteur de confiance.

### Pour les acheteurs
- **Paiement fractionné** (3x sans frais) : augmente le pouvoir d’achat.
- **Frais de livraison transparents** : forfait par zone géographique (ex: 5 000 FCFA Douala, 10 000 FCFA Yaoundé, 25 000 FCFA Abidjan).
- **Assurance premium optionnelle** : rassure sur les achats de valeur.

### Avantage concurrentiel
- Commission **15–25 %** inférieure aux galeries physiques (40–50 %) et comparable aux meilleures marketplaces (Afrikanizm 15 %, Latitudes 30 %).
- Valeur ajoutée : certificat blockchain, logistique dédiée, curation locale.

## 9. Résumé exécutif

Le modèle économique de SANKOFA est **viable à moyen terme** (rentabilité atteinte vers **80 commandes/mois**). Il repose sur une commission dégressive, des services optionnels et une maîtrise rigoureuse des coûts variables. La première année est déficitaire (investissement), mais la croissance du GMV permet d’atteindre l’équilibre en année 2. Les marges élevées sur les services additionnels (certificat blockchain, mise en avant) accélèrent la rentabilité. Le financement initial nécessaire est de **15–30 M FCFA** pour le lancement.
